1. The JetOverlay (The Floating Surface)
This is not an app screen; it is your persistent system utility.

Idle State: A small, semi-transparent circular FAB (Floating Action Button) snapped to the edge of the screen. Material 3 tonal styling.

Drag State: Opaque, follows the finger. Screen edges show subtle drop zones to dismiss or pin.

Active State (The Tap): The bubble instantly expands into a compact, elevated card (The Capture Sheet).

Visual Cue: A subtle colored dot on the edge of the bubble indicates if the background service is healthy (Green) or if the OS killed it (Red).

2. The Capture Sheet (The Core MVP Deliverable)
This is the UI that appears over whatever app the user is currently in when they tap the bubble. It must be operated easily with one hand.

Top Bar: Tiny Capsule logo, source attribution (e.g., "Sourced from Chrome"), and a timestamp.

Content Body: A scrollable text field displaying the newly captured clipboard data or explicit share text.

Action Row (Horizontal): * Save & Close (Primary, high emphasis)

Tag (Secondary)

Summarize (Secondary)

Discard (Icon button, low emphasis)

Feedback Footer: A tiny status text indicating model processing (e.g., "Local SLM generating tags...").

3. The Main Workspace (The Inbox)
This is what the user sees when they launch the actual Capsule app from their app drawer. It is a timeline of their digital memory.

Top App Bar: Global search bar (triggering local vector search) with a filter icon (filter by source app, date, or tag).

Main Feed: A vertical LazyColumn of Memory Cards.

Memory Card: Displays a generated title/summary, a snippet of the raw text, the source icon, and auto-tags.

Bottom FAB: To manually ingest a thought or photo without using the overlay.

4. The Memory Detail View
Accessed by tapping a Memory Card in the Inbox. This is where the user interacts with the AI agent regarding a specific piece of context.

Header: Source, Timestamp, and Tags (editable).

Content Split:

Top Half: The raw captured text/image (collapsible).

Bottom Half: The AI-generated summary or extraction.

Bottom App Bar / Chat Input: A text input field to query the agent specifically about this memory (e.g., "Extract the action items from this text").

5. Settings & System Health (Crucial for Android)
Because you are fighting the OS, you must expose system health to the user.

Service Dashboard: A clear indicator showing if the CapsuleOverlayService is active. Includes a direct deep-link button to the device's specific battery optimization settings to whitelist the app.

Privacy & Source Allowlist: Toggle switches for capturing clipboard, and a list of installed apps where the user can disable the overlay from appearing (e.g., disabling Capsule over their banking app).

Intelligence Tier: A selector for the AI backend.

Option A: Fast/Local (Raw SQLite search, no SLM).

Option B: Maximum Privacy (ONNX local SLM inference - heavy battery use).

Option C: Hybrid Cloud (Sends data to external API for heavy processing).
6. The Global Agent Chat (The Synthesis Engine)
Unlike the Memory Detail View (which chats about one specific item), this is your primary interface for querying your entire digital life. It requires a dedicated, full-screen UI.

Top Bar: Mode Selector toggle.

Local Vault Mode (Queries only your saved context using RAG).

Internet + Vault Mode (Agents can query the web and cross-reference your notes).

The Chat Feed: Standard message bubbles, but with a critical PKM requirement: Citation Pills. When the agent answers based on your data, it must generate clickable pill-shaped tags at the bottom of the message (e.g., [Chrome: Project Spec], [Clipboard: Nov 12]).

Agentic Thought Transparency: An expanding accordion above the agent's final answer showing its LangChain workflow steps (e.g., Thinking: Querying Vector DB... Extracting keywords... Synthesizing). This builds trust.

7. The Knowledge Connections (The Neural Map)
Recall touts a "Knowledge Graph," but rendering a massive 3D node-graph on a 6-inch mobile screen is a notorious UX nightmare. You need a mobile-first way to show relationships.

Bidirectional Link Lists: At the bottom of every Memory Card, instead of a visual web, use a clean list of "Related Context" generated by your vector similarity search.

Auto-Link Formatting: If a user is viewing a note about "Overclock," the UI should automatically highlight other mentions of that entity across the vault, allowing the user to tap the word and instantly pull up a bottom sheet of related memories. It mimics the seamless, structured linkage found in advanced study tools without the visual clutter.

8. Augmented Browsing (The Contextual Whisper)
This is how you beat the friction of manual search.

The "Deja Vu" Indicator: When the JetOverlay bubble is active over Chrome, and the URL or screen text triggers a high-confidence semantic match in your local vector database, the bubble pulses or briefly expands to say: "3 related notes found."

The Peek Sheet: Tapping that pulsing bubble slides up a half-screen sheet showing the relevant past memories, giving you instant context on the article you are currently reading without leaving the browser.

9. Spaced Repetition / The Quiz Interface
To ensure long-term retention of captured data.

Daily Review Deck: A swipeable card interface (Tinder-style UX) that surfaces old, highly-rated notes or AI-generated questions based on your saves.

Card Anatomy: Front of the card shows a concept or AI-generated question; tapping flips the card to reveal the source text and AI summary. Swipe right to increase the interval, swipe left to review again soon.
